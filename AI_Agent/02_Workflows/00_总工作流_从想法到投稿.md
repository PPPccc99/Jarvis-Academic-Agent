# 总工作流：从一个小思路到投稿论文

## 前置步骤 A：领域适配

用户使用贾维斯时，研究主题、专业方向和论文类型可能完全不同。主控智能体必须先读取并执行：

```text
07_跨学科领域适配与材料清单工作流.md
```

先建立本轮领域画像、论文类型、证据体系和材料缺口，再决定是否进入文献拆解、选题、写作、验证或润色。

## 前置步骤 B：进度识别

用户使用贾维斯时，不一定从初始想法开始。主控智能体必须继续读取并执行：

```text
05_论文进度识别与最小调用工作流.md
```

先判断论文处于 P0-P7 或 PX 阶段，再只调用必要子智能体。若用户已经写完引言、前三章或全文初稿，不默认从选题阶段重来。

## 阶段 0：项目初始化

目标：建立本项目长期可复用知识库，但知识库只服务本项目、本主题或本领域，不在跨项目同步时默认复制。

```text
用户提供资料目录
-> 主控智能体建立任务
-> 文件提取与资料整理智能体生成文件索引
-> 目标期刊论文拆解智能体拆解用户指定目标期刊
-> 参考文献论文拆解智能体拆解研究现状
-> 文献与证据库管理智能体建立证据库
```

输出：

- 目标期刊写法规律
- 参考文献技术现状
- 证据条目库
- 文件索引

## 阶段 1：研究想法评估

目标：判断用户初步想法是否值得发展为论文。

```text
用户初始想法
-> 主控智能体整理问题
-> 选题与创新点写作智能体初评
-> 参考文献证据调用
-> 目标期刊偏好调用
-> 审稿人模拟智能体审查
-> 主控智能体给出推进/调整/暂缓判断
```

通过条件：

- 问题具体
- 与已有研究有区别
- 有可能获得数据支撑
- 工程意义不空泛

硬停止条件：

- 审稿人模拟智能体判定“审稿结论：阻塞”。
- 审稿人模拟智能体判定“是否允许进入下一阶段：否”。
- 选题无法形成可验证创新点。
- 用户当前条件无法支撑必要仿真、实验或数据验证。
- 算法复杂度或硬性条件明显超过现场可接受范围，且不能说明足够工程收益。

出现硬停止条件时，主控智能体不得进入任务书、正文、仿真计划或全文润色阶段，只能输出不可行原因、缺失条件和可替代方向。

## 阶段 2：选题和创新点确定

目标：形成题目、科学问题、技术问题和创新点。

输出至少包括：

- 题目候选 3-5 个
- 推荐题目 1 个
- 研究问题 1-2 个
- 创新点 2-4 条
- 需要支撑的仿真/实验清单
- 可能拒稿风险

审稿人模拟智能体必须审查。若审查不通过，最多修改 3 轮；若审稿人模拟智能体直接判定选题不具备论文可行性，则立即停止后续写作流程。

## 阶段 3：论文任务书

目标：确定论文整体结构和每章任务。

工程/实验/仿真类论文可优先采用六章结构：

```text
第一章 引言
第二章 理论基础、问题建模或总体框架
第三章 核心方法、算法、传感结构或系统设计
第四章 仿真建模与分析
第五章 实验平台与结果验证
第六章 结论
```

其他论文类型应按目标期刊近期论文和领域惯例调整。例如算法/模型论文可采用“引言-相关工作-方法-实验设置-结果与讨论-结论”，综述论文可采用“引言-检索策略-分类框架-主题分析-挑战与趋势-结论”，理论论文可采用“引言-假设与模型-推导-性质分析-验证或案例-结论”。不得把工程类六章结构强行套用到所有学科。

通过条件：

- 每章服务于明确问题
- 每个创新点都有对应章节支撑
- 第四章和第五章能形成验证闭环
- 第六章结论不会超出结果范围

## 阶段 4：正文前半部分写作

目标：完成摘要初稿、引言、第二章、第三章。

注意：

- 摘要如结果未完成，只写结构性初稿。
- 引言必须体现背景、不足、问题、方法和贡献。
- 第二章和第三章不得提前写入未验证结果。

## 阶段 5：仿真和实验计划

目标：把创新点转化为可执行验证计划。

必须明确：

- 验证目的
- 缺陷模型或实验对象
- 软件、设备和参数。CST、MATLAB、COMSOL、TracePro、SolidWorks、有限元软件仅作为工程类示例
- 若非工程类论文，替换为本领域可用工具、数据源、平台、样品或实验环境
- 对照组
- 评价指标
- 数据记录方式
- 安全和工程约束
- 涉及算法时的复杂度、部署条件和现场可用性

## 阶段 6：结果校核和分析

目标：确认结果能否进入论文，并撰写结果章节。

```text
用户提供结果
-> 数据与结果校核智能体检查
-> 可用结果进入结果分析
-> 图表呈现智能体规划图表
-> 仿真实验结果分析智能体撰写第四、五、六章
-> 审稿人模拟智能体审查结果支撑
```

## 阶段 7：全文整合和润色

目标：形成完整论文终稿。

```text
正文各章合并
-> 语言与格式润色智能体
-> 图表呈现智能体复核
-> 投稿规范检查智能体
-> 审稿人模拟智能体终审
-> 主控智能体输出投稿前问题清单
```

## 阶段 8：投稿前检查

目标：降低形式问题和审稿风险。

检查：

- 题目、摘要、关键词
- 作者信息和双盲要求
- 图表、公式、参考文献
- 结论强度
- 工程意义
- 局限性
- 投稿材料清单

若存在证据不足，不通过语言润色掩盖，必须向用户说明。
