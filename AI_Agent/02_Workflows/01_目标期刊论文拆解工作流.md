# 目标期刊论文拆解工作流

## 适用场景

用户已经指定目标期刊名称，并提供该目标期刊近期发表论文所在文件夹的详细本地地址。

```text
目标期刊名称：由用户提供
目标期刊近期论文文件夹：由用户提供详细本地地址
```

## 执行原则

1. 第一次执行做全量拆解。
2. 后续若目标期刊不变且未新增论文，不重复全量拆解。
3. 新增论文只做增量拆解。
4. 拆解结果用于学习期刊写法，不用于复制原文或替代技术创新。
5. 未获得目标期刊名称和目标期刊近期论文文件夹前，不开始拆解，也不在项目中盲目搜索。

## 步骤

### 1. 建立论文清单

记录：

- 文件名
- 文件路径
- 文件格式
- 题名
- 作者
- 年份
- 研究方向
- 是否成功提取文本
- 是否完成拆解

### 2. 文本提取

优先级：

```text
Word -> PDF -> 可提取文本的 CAJ -> 用户转换后的文本
```

CAJ 无法可靠提取时，记录失败原因并请求用户转换。

### 3. 单篇论文拆解

每篇论文拆解以下维度：

- 选题类型
- 题目结构
- 摘要目的、方法、结果、结论
- 引言逻辑链
- 章节结构
- 创新点类型
- 技术路线深度
- 图表类型和插入位置
- 仿真/实验结果分析方式
- 对比对象
- 参考文献结构
- 语言风格
- 可借鉴点
- 不可照搬点

### 4. 横向归纳

对多篇论文进行横向比较：

- 高频题目表达方式
- 高频引言组织方式
- 常见验证组合
- 常见图表顺序
- 常见创新点表达
- 常见工程意义表达
- 常见结论边界

### 5. 生成知识库

输出到：

```text
03_Knowledge_Base\target_journal\{目标期刊名}\
```

并生成：

- 写作规律总表
- 题目摘要引言规律
- 图表与结果规律
- 目标期刊可借鉴表达清单
- 目标期刊不宜模仿问题清单

## 质量要求

目标期刊拆解不得只给笼统结论。每条规律都要指向至少一篇来源论文编号。

## 后续调用

选题、摘要、引言、图表、结果分析和投稿检查阶段，均可读取本工作流生成的知识库摘要。
