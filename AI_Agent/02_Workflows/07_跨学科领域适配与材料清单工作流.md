# 跨学科领域适配与材料清单工作流

## 目的

让贾维斯在不同主题、不同专业、不同研究方向下工作时，先识别本轮论文的领域规则、论文类型、证据体系和材料缺口，避免把一个项目的专业假设套用到另一个项目。

## 触发条件

每次用户说“召唤贾维斯”后，主控智能体必须执行本工作流。若用户已经在同一对话中提供过领域信息，可复用并简要确认；新对话必须重新确认。

## 步骤 1：建立领域画像

主控智能体先确认：

```text
研究领域：
专业方向：
核心研究对象：
应用场景：
论文类型：
目标期刊：
当前进度：
本轮任务目标：
```

若用户无法一次性提供完整信息，可以先记录“不确定”，但不得在后续输出中把不确定内容写成事实。

## 步骤 2：识别证据体系

按论文类型确定最低证据要求：

| 论文类型 | 最低证据要求 |
|---|---|
| 实验研究 | 原始数据、样品/设备/工况、重复性或误差说明、对照组或基准条件 |
| 仿真研究 | 模型文件或建模说明、参数来源、边界条件、验证或敏感性分析 |
| 算法/模型研究 | 数据来源、评价指标、基线方法、消融或参数分析、复杂度与可复现性 |
| 理论研究 | 假设边界、变量定义、推导过程、与已有理论关系、可验证示例 |
| 工程验证 | 现场或近现场条件、部署约束、安全可靠性、成本或维护边界 |
| 器件/系统设计 | 结构参数、制造/实现条件、性能测试、对比对象、失效或局限分析 |
| 案例研究 | 案例来源、选择理由、资料可信度、分析框架、可推广边界 |
| 综述论文 | 检索数据库、关键词、时间范围、纳排标准、分类框架、偏倚说明 |

若论文属于混合类型，应合并对应证据要求。

## 步骤 3：盘点可用材料

主控智能体生成材料清单：

```text
目标期刊近期论文：
参考文献：
论文草稿：
真实数据：
仿真模型/结果：
实验记录/报告：
图表/图片：
代码/算法流程：
设备/软件/样品条件：
投稿模板/作者指南：
标准/规范/伦理/安全/保密约束：
导师/审稿意见：
```

材料状态分为：

```text
已提供 / 可提供但未上传 / 暂无 / 不适用 / 需要用户确认
```

## 步骤 4：判断本轮可推进范围

主控智能体根据材料完整性判断本轮最多能推进到哪里：

```text
仅能做构思建议
可做选题和创新点初评
可做任务书和结构设计
可做章节草稿
可做仿真/实验计划
可做结果分析
可做全文体检和润色
可做投稿前检查
```

若缺少关键证据，必须明确说明不能推进的阶段。例如没有真实结果时，不得写确定性结果和结论；没有设备条件时，不得给出不可执行实验计划；没有目标期刊规律时，不得声称完全符合期刊偏好。

## 步骤 5：最小调用原则

领域适配阶段默认只调用：

```text
01 主控智能体
15 审稿人模拟智能体，必要时
02 文件提取与资料整理智能体，只有用户提供材料路径时
03 目标期刊论文拆解智能体，只有需要期刊规律时
04 参考文献论文拆解智能体，只有需要研究现状支撑时
```

不得因为领域未知而调用全部 15 个智能体。领域画像完成后，再进入进度识别和最小调用工作流。

## 输出格式

主控智能体应输出：

```text
领域画像：
论文类型：
当前进度：
已确认材料：
关键缺口：
本轮最多可推进到：
本轮建议调用智能体：
需要用户补充：
```

该输出应保存到本轮工作文件夹的 `01_任务记录` 子目录。
