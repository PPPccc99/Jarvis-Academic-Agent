# 审稿循环工作流

## 目的

在关键阶段提前暴露拒稿风险，避免到终稿阶段才发现选题、证据或结构问题。

## 触发节点

必须触发审稿人模拟智能体的节点：

1. 选题与创新点确定后。
2. 论文任务书完成后。
3. 正文前三章完成后。
4. 仿真和实验计划完成后。
5. 结果分析章节完成后。
6. 全文润色前。
7. 投稿前终审。

## 选题阶段质量门禁

选题与创新点阶段是硬门禁。审稿人模拟智能体必须判断该思路是否具备发展成论文的基本可行性。

若出现以下任一结论：

```text
审稿结论：阻塞
是否进入下一阶段：否
```

主控智能体必须停止后续流程，不得继续调用论文任务书写作智能体、正文写作智能体、仿真和实验计划安排智能体、语言润色智能体。

停止后只允许输出：

- 不可行原因
- 主要证据缺口
- 缺失实验/仿真/数据条件
- 可补救路径
- 可替代研究方向

## 审稿输出格式

```text
审稿结论：通过 / 修改后通过 / 暂缓推进 / 阻塞
总体判断：
是否进入下一阶段：是 / 否
主要问题：
拒稿风险：
必须修改：
建议修改：
可不修改：
需要用户补充的信息：
```

## 迭代规则

1. 同一阶段最多审查 3 轮。
2. 第 1 轮发现结构性问题时，优先修改结构。
3. 第 2 轮仍发现证据不足时，优先请求用户补充材料。
4. 选题阶段第 1 轮若已明确不可行，可直接阻塞，无需凑满 3 轮。
5. 第 3 轮仍不通过时，停止循环，由主控智能体给出阻塞总结。

## 主控裁决

审稿人模拟智能体只提供意见。是否采纳由主控智能体裁决，并记录：

- 采纳意见
- 部分采纳意见
- 未采纳意见
- 未采纳原因

## 防止过度审查

审稿意见必须与目标期刊、研究方向和当前阶段相关。不得在选题阶段要求完整实验结果，不得在语言润色阶段重新推翻已确认且有证据支撑的技术路线。
