# 论文进度识别与最小调用工作流

## 目的

用户使用贾维斯时，不一定从“一个小思路”开始。可能已经有题目、引言、前三章、仿真计划、实验结果、全文初稿或投稿稿。主控智能体必须先识别论文进度，再只调用必要子智能体，避免重复工作和 token 浪费。

## 总原则

1. 不默认从选题阶段重来。
2. 不默认调用全部 15 个智能体。
3. 先判断“已有材料”和“当前目标”，再分配任务。
4. 已经完成且用户未要求重写的部分，只做必要审查，不全文重写。
5. 长文档先提取目录、摘要、章节标题、图表清单和关键段落，再决定是否深读。

## 启动时必须询问或识别的信息

若用户未说明，主控智能体应尽量从文件和用户描述中判断：

```text
1. 研究领域、专业方向和核心研究对象是什么
2. 论文类型：实验 / 仿真 / 算法或模型 / 理论 / 工程验证 / 器件或系统设计 / 案例研究 / 综述 / 混合类型
3. 当前已有材料：只有想法 / 有题目 / 有引言 / 有前三章 / 有验证计划 / 有结果 / 有全文初稿 / 已接近投稿
4. 当前目标：构思 / 补强逻辑 / 写某章 / 修改全文 / 分析结果 / 投稿前检查
5. 目标期刊是什么
6. 目标期刊近期论文文件夹详细本地地址是什么
7. 参考文献文件夹详细本地地址是什么
8. 是否已有真实数据、仿真、实验、案例、代码、模型或推导过程
9. 用户是否希望重写，还是只审查和局部修改
10. 是否已经有导师意见、审稿意见、批注文件或返修意见需要处理
11. 是否存在伦理、安全、保密、标准规范或设备条件限制
```

## 进度级别判定

| 进度级别 | 判定特征 | 主控判断 |
|---|---|---|
| P0 初始想法 | 只有研究方向、技术想法或零散笔记 | 从选题和创新点进入 |
| P1 已有题目/创新点 | 有题目、摘要草稿或创新点列表，但结构未定 | 重点审查选题、创新点和任务书 |
| P2 已有引言 | 引言已写出，但方法章节未完整 | 重点审查引言逻辑和章节规划 |
| P3 已有前三章 | 摘要/引言/理论/方法章节基本完成 | 不重写前三章，先审查逻辑和证据，再进入仿真/实验计划 |
| P4 已有仿真/实验计划 | 已有验证方案，但结果未完成 | 重点检查验证方案能否支撑创新点 |
| P5 已有仿真/实验结果 | 有数据、报告、图片或波形 | 先数据校核，再结果分析和图表组织 |
| P6 已有全文初稿 | 六章或完整论文初稿已成形 | 先全文结构体检，再局部修订、润色和投稿规范检查 |
| P7 投稿前稿件 | 内容基本完成，只需投稿前处理 | 只调用润色、投稿规范、审稿人终审和必要图表检查 |
| PX 阻塞/返工 | 数据不支撑结论、创新点不成立、结构严重断裂 | 主控输出阻塞原因，请用户决策 |
| PR 外部意见处理 | 用户提供导师意见、审稿意见、批注或返修意见 | 先生成采纳矩阵，再局部调用相关智能体 |

## 最小调用组合

| 进度 | 最小调用智能体 | 可选调用 | 不应默认调用 |
|---|---|---|---|
| P0 初始想法 | 01 主控、06 选题创新点、15 审稿人 | 03 目标期刊、04 参考文献、05 证据库 | 08 正文、10 数据校核、13 润色 |
| P1 已有题目/创新点 | 01、06、07 任务书、15 | 03、04、05 | 08 全文写作、13 全文润色 |
| P2 已有引言 | 01、08 正文写作、15 | 05、13 局部润色、03 | 09、10、11，除非涉及验证内容 |
| P3 已有前三章 | 01、08 局部审查、09 仿真实验计划、15 | 05、12 图表 | 06 重新选题，除非创新点不成立 |
| P4 已有验证计划 | 01、09、10 预校核、15 | 12 图表、04 对比文献 | 08 大规模重写 |
| P5 已有结果 | 01、10 数据校核、11 结果分析、12 图表、15 | 13 局部润色 | 06 选题重做，除非结果推翻创新点 |
| P6 全文初稿 | 01、15 全文体检、13 润色、14 投稿规范 | 10 数据抽查、12 图表复核、05 证据核查 | 03/04 全量拆解，除非知识库缺失 |
| P7 投稿前稿件 | 01、13、14、15 | 12 图表终检 | 06、07、08 大规模重写 |
| PX 阻塞/返工 | 01、15、相关问题智能体 | 05 证据库 | 无限制，但必须说明原因 |
| PR 外部意见处理 | 01、15、对应问题智能体 | 02 文件提取、13 润色、14 投稿规范 | 不直接全稿重写 |

## 材料读取策略

### Word 或 PDF 稿件

默认只读取：

- 题目
- 摘要
- 目录或一级/二级标题
- 引言首尾段
- 每章小结或关键段落
- 图表题名
- 结论

只有发现逻辑断裂、证据不足或用户要求细改时，才深读相关章节。

### 已有前三章

不默认重写。主控应先让正文写作智能体输出：

```text
章节结构是否合理：
引言逻辑链是否完整：
第二章和第三章是否支撑创新点：
哪些段落需要局部改写：
是否可以进入仿真/实验计划：
```

### 已有全文初稿

不直接润色全文。先做全文体检：

```text
题目-摘要-创新点是否一致：
章节结构是否闭环：
图表是否支撑结果：
数据是否支撑结论：
结论是否越界：
投稿格式是否存在硬伤：
```

体检后再决定是否调用语言润色智能体。

## 主控输出格式

每次启动贾维斯后，主控智能体应先输出简短判断：

```text
识别进度：
当前目标：
本轮调用智能体：
暂不调用智能体：
原因：
需要用户补充：
下一步：
```

若识别到 PR 外部意见处理，还必须输出：

```text
意见来源：
意见文件：
意见条目数量：
本轮处理范围：
是否需要用户确认真实数据或导师偏好：
```

## Token 节省规则

1. 同一轮最多默认调用 3-5 个子智能体。
2. 已有全文初稿时，先审查结构摘要，不直接全稿重写。
3. 已有结果时，优先校核数据，不先润色语言。
4. 投稿前任务不重新拆解目标期刊论文，除非目标期刊知识库不存在或用户新增了近期论文。
5. 审稿人模拟智能体只审当前阶段，不默认审全文。
