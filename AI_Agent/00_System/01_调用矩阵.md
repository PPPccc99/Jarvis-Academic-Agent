# 智能体调用矩阵

## 阶段调用

| 阶段 | 主调用智能体 | 必要协作智能体 | 输出 |
|---|---|---|---|
| 领域适配与材料盘点 | 主控智能体 | 必要时文件提取、审稿人模拟 | 领域画像、论文类型、证据体系、材料缺口 |
| 进度识别 | 主控智能体 | 必要时文件提取、审稿人模拟 | P0-P7/PX 进度判断和最小调用计划 |
| 初始化 | 主控智能体 | 文件提取、目标期刊拆解、参考文献拆解、证据库管理 | 项目资料索引、知识库初版 |
| 目标期刊拆解 | 目标期刊论文拆解智能体 | 文件提取、证据库管理 | 目标期刊写法规律 |
| 研究现状梳理 | 参考文献论文拆解智能体 | 文件提取、证据库管理 | 技术现状和研究空白 |
| 选题与创新点 | 选题与创新点写作智能体 | 证据库管理、目标期刊拆解、参考文献拆解、审稿人模拟 | 题目、科学问题、创新点；若不可行则输出阻塞原因 |
| 论文任务书 | 论文任务书写作智能体 | 审稿人模拟、主控 | 章节结构、研究路线、写作任务 |
| 摘要和前三章 | 正文写作智能体 | 证据库管理、审稿人模拟、语言润色 | 正文初稿 |
| 仿真和实验计划 | 仿真和实验计划安排智能体 | 审稿人模拟、数据校核 | 仿真/实验方案 |
| 结果分析 | 仿真实验结果分析智能体 | 数据校核、图表呈现、审稿人模拟 | 结果章节、结论初稿 |
| 图表组织 | 图表呈现智能体 | 数据校核、目标期刊拆解 | 图表清单、插入位置、图题建议 |
| 终稿润色 | 语言与格式润色智能体 | 投稿规范检查、审稿人模拟 | 终稿润色版 |
| 投稿前检查 | 投稿规范检查智能体 | 审稿人模拟、主控 | 投稿检查清单 |
| 导师/审稿意见处理 | 主控智能体 | 文件提取、审稿人模拟、对应问题智能体 | 意见采纳矩阵、局部修改方案、处理记录 |

## 插件和技能调用

| 任务类型 | 优先技能/工具 | 说明 |
|---|---|---|
| 论文技术审查 | paper-review-loop | 用于阶段性审稿和拒稿风险分析 |
| PDF 提取与检查 | pdf | 用于 PDF 文献文本、图表、页面基础检查 |
| Word 文档读写 | documents/doc | 默认低 token 模式，不做完整渲染 |
| Excel/CSV 表格 | spreadsheets | 用于拆解表、证据表、数据表 |
| Zotero 文献库 | zotero | 仅当用户启用 Zotero 或要求导出引用时调用 |
| 浏览器检查 | browser | 仅当需要检查网页、本地系统或在线投稿页面时调用 |
| 图像生成 | imagegen | 仅当用户需要示意图、概念图、视觉素材时调用；论文数据图优先基于真实数据绘制 |

## 调用约束

1. 主控智能体是唯一任务分发入口。
2. 子智能体不得主动越权调用无关阶段的智能体。
3. 审稿人模拟智能体只给意见，不直接改写全文；是否采纳由主控裁决。
4. 数据与结果校核智能体可以否决结果分析智能体的写作请求。
5. 语言与格式润色智能体不得新增技术结论。
6. 启动贾维斯后必须先做领域适配和进度识别，再按最小调用组合调度子智能体。
7. 选题与创新点阶段若被审稿人模拟智能体判定不可行，主控智能体必须停止后续流程，不得继续写任务书或正文。
8. 导师/审稿意见不得直接全量照改，必须先形成采纳矩阵，再调用对应子智能体局部处理。
