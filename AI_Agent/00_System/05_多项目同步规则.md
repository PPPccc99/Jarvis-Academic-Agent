# 多项目贾维斯同步规则

## 核心原则

当用户要求在两个项目之间同步贾维斯更新时，默认只同步“通用能力和规则”，不得同步任何项目独有知识、任务输出或用户论文材料。

同步前必须区分：

```text
通用配置：可以同步
项目知识：默认不同步
任务输出：默认不同步
运行状态：默认不同步
```

## 默认允许同步

以下内容属于通用能力，用户要求“同步贾维斯规则/功能/智能体更新”时可以同步：

```text
README.md
00_System 中的全局规则、调用矩阵、命名规范、启动说明、可靠性设计、同步规则和脚本
01_Agents 中的智能体功能、边界、调用关系和质量门槛
02_Workflows 中的工作流
05_Templates 中的通用模板
07_Dashboard 中的页面代码、样式、前端脚本、状态更新脚本、启动脚本和停止脚本
```

同步后必须把文件中的源项目根路径替换为目标项目路径。例如从 `D:\SourceProject` 同步到 `D:\TargetProject` 时，目标项目文件内应统一使用 `D:\TargetProject`。在模板仓库中，路径占位符为 `{{PROJECT_ROOT}}`，安装后由 `install.ps1` 替换为实际项目路径。

## 默认禁止同步

以下内容属于项目独有内容，除非用户明确点名要求，否则不得同步：

```text
03_Knowledge_Base
04_Task_Records
06_Outputs
目标期刊论文原始文件
参考文献原始文件
目标期刊拆解结果
参考文献拆解结果
证据库条目
用户论文草稿、数据、图表、代码、仿真模型、实验报告
导师/审稿意见原文及采纳矩阵
年.月.日（序号）_贾维斯智能体工作任务 文件夹
07_Dashboard\agent_status.json
07_Dashboard\dashboard_server.pid
临时文件、缓存、日志和本地服务运行状态
```

## 同步流程

1. 明确源项目路径和目标项目路径。
2. 只复制允许同步的通用配置文件和目录。
3. 排除所有知识库、任务记录、输出、用户材料和运行状态。
4. 将目标项目中的路径文本替换为目标项目路径。
5. 检查目标项目中是否残留源项目路径。
6. 不自动启动目标项目仪表盘，除非用户明确要求。
7. 向用户报告同步范围、未同步内容和需要重新提供的项目材料。

## 冲突处理

若目标项目已经对同一通用规则做过本项目专属修改，主控智能体不得直接覆盖，应先说明冲突并请求用户决定。若只是路径差异、仪表盘端口或项目名差异，可自动保留目标项目设置。

## 用户提示语解释

当用户说“把这里对贾维斯的修改同步到另一个项目”时，默认解释为：

```text
同步通用规则、智能体配置、工作流、模板和仪表盘代码。
不复制本项目知识库、任务结果、目标期刊拆解、参考文献拆解、论文草稿、数据或运行状态。
```
