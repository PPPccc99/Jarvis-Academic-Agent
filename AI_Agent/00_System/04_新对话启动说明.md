# 新对话启动说明

## 一、正式名称

本项目的论文写作多智能体体系正式命名为：

```text
贾维斯
```

含义：以主控智能体为调度中心，长期调用 15 个论文写作子智能体，辅助不同主题、不同专业、不同研究方向的高水平学术期刊正刊论文，完成从领域识别、论文构思、文献拆解、选题创新点、任务书、正文、仿真/实验计划、结果分析、图表、润色、投稿检查到审稿模拟的完整流程。

## 二、默认启动语句

用户在任意新对话或当前对话中说出：

```text
召唤贾维斯
```

即表示启动 `{{PROJECT_ROOT}}` 项目的论文写作多智能体体系。

该语句触发后，Codex 应执行：

1. 进入或确认项目路径：`{{PROJECT_ROOT}}`。
2. 立即在项目主文件夹下创建本次对话专用工作文件夹，命名规则为：`年.月.日（序号）_贾维斯智能体工作任务`。
3. 本次对话中所有新生成或处理得到的文件，默认全部保存到该工作文件夹或其子目录。
4. 读取 `{{PROJECT_ROOT}}\AI_Agent\README.md`。
5. 读取 `{{PROJECT_ROOT}}\AI_Agent\00_System\00_全局规则.md`。
6. 读取 `{{PROJECT_ROOT}}\AI_Agent\00_System\06_引言与正文反模板化写作规则.md`。
7. 读取 `{{PROJECT_ROOT}}\AI_Agent\00_System\01_调用矩阵.md`。
8. 读取 `{{PROJECT_ROOT}}\AI_Agent\00_System\03_运行模型与可靠性设计.md`。
9. 读取 `{{PROJECT_ROOT}}\AI_Agent\02_Workflows\00_总工作流_从想法到投稿.md`。
10. 读取 `{{PROJECT_ROOT}}\AI_Agent\02_Workflows\05_论文进度识别与最小调用工作流.md`。
11. 读取 `{{PROJECT_ROOT}}\AI_Agent\02_Workflows\07_跨学科领域适配与材料清单工作流.md`。
12. 若本轮涉及跨项目同步，读取 `{{PROJECT_ROOT}}\AI_Agent\00_System\05_多项目同步规则.md`。
13. 默认启动并打开可视化仪表盘，同步将主控智能体状态更新为“启动信息收集中”。
14. 立即按“三、启动后必须确认的信息”向用户一次性提问。
15. 等待用户提供研究领域、目标期刊、资料文件夹和当前任务信息；若用户已在同一句话中提供部分信息，只追问缺失项。
16. 收到用户信息后，由主控智能体建立本轮领域画像、材料清单和缺口清单。
17. 判断用户当前任务阶段，并按最小调用组合读取对应子智能体配置文件。
18. 后续全程按主控智能体调度流程参与本对话中的论文工作。

可使用脚本创建工作文件夹：

```powershell
python {{PROJECT_ROOT}}\AI_Agent\00_System\scripts\create_jarvis_task_folder.py --root {{PROJECT_ROOT}}
```

## 三、启动后必须确认的信息

“召唤贾维斯”后，主控智能体必须先向用户确认以下信息，不得默认任何目标期刊、固定专业方向或固定资料目录，也不得在本地项目中盲目搜索参考文献目录。

启动提问必须与可视化仪表盘启动同步进行：仪表盘可以先打开并显示“启动信息收集中”，随后主控智能体一次性向用户提出以下问题。若用户在“召唤贾维斯”同一句话中已经提供了某些答案，则只追问缺失信息，不重复询问已明确的信息。

必须询问：

```text
1. 本轮论文的研究领域、专业方向和核心研究对象是什么？
2. 论文类型更接近哪一种：实验、仿真、算法/模型、理论、工程验证、器件/系统设计、案例研究、综述，还是混合类型？
3. 你的目标期刊是什么？
4. 你已经下载好的目标期刊近期论文放在哪个文件夹？请提供该文件夹的详细本地地址。
5. 你的参考文献都下载到了哪个文件夹？请提供参考文献文件夹的详细本地地址。
6. 你的论文目前进度是什么？例如只有想法、已有题目、已有引言、已有前三章、已有结果、已有全文初稿或投稿前稿件。
7. 本轮希望贾维斯优先完成什么任务？
8. 除目标期刊论文和参考文献外，你还有哪些材料可以提供：草稿、数据、图表、代码、仿真模型、实验报告、设备清单、投稿模板、导师/审稿意见、标准规范或伦理/保密要求？
```

若用户本轮任务暂时不需要目标期刊拆解或参考文献拆解，主控智能体可以先记录“本轮暂不需要”，但不得自行假设文件夹路径。

若用户已经在同一对话中提供过上述信息，主控智能体应复用本轮已确认路径，不重复询问；若开启新对话，则应重新确认。

## 四、可视化仪表盘默认打开

从本约定开始，只要用户说：

```text
召唤贾维斯
```

Codex 应默认启动本地仪表盘服务并打开：

```text
http://127.0.0.1:8765/index.html
```

若端口 `8765` 已被占用，允许自动使用 `8766-8775` 范围内的可用端口，并告知用户实际地址。

若仪表盘服务已在运行，则直接打开已有地址，不重复启动。

## 五、手动启动和停止仪表盘

用户也可以手动运行：

```powershell
{{PROJECT_ROOT}}\AI_Agent\07_Dashboard\start_dashboard.ps1
```

停止仪表盘：

```powershell
{{PROJECT_ROOT}}\AI_Agent\07_Dashboard\stop_dashboard.ps1
```

## 六、启动后的工作方式

“召唤贾维斯”后，主控智能体应全程参与当前对话的论文写作流程，包括：

1. 先创建本次对话专用工作文件夹。
2. 同步启动并打开可视化仪表盘。
3. 同步向用户一次性确认研究领域、论文类型、目标期刊、目标期刊近期论文文件夹、参考文献文件夹和可用支撑材料。
4. 收到用户信息后，建立本轮领域画像和材料缺口清单。
5. 判断当前任务属于构思、拆解、写作、仿真、实验、结果分析、润色还是投稿检查。
6. 分配任务给对应子智能体。
7. 更新 `agent_status.json`，使可视化界面显示当前调用链。
8. 汇总子智能体结论。
9. 在资料不足、逻辑冲突、证据不足、伦理/安全/保密或工程条件不明时向用户提问。
10. 记录关键阶段输出和审稿意见。
11. 若用户提供导师意见、审稿意见或返修意见，先生成采纳矩阵，再调用对应子智能体局部处理。

## 七、用户示例

```text
召唤贾维斯。研究方向是某专业的某研究对象，论文类型偏实验和仿真结合。目标期刊是《某期刊》。目标期刊近期论文在 D:\...\目标期刊论文。参考文献在 D:\...\参考文献。我现在只有一个初步思路，请先帮我判断选题、创新点和可行性。
```

主控智能体应调用：

- 选题与创新点写作智能体
- 参考文献论文拆解智能体，若需要研究现状支撑
- 目标期刊论文拆解智能体，若需要期刊偏好支撑
- 审稿人模拟智能体，审查选题风险

## 八、注意事项

1. 贾维斯不是 15 个常驻后台模型进程，而是长期保存的多智能体配置、工作流和可视化调度体系。
2. 每次新对话需要通过“召唤贾维斯”让 Codex 读取本项目配置。
3. 可视化状态依赖 `agent_status.json`，只有主控智能体或状态脚本更新后才会变化。
4. 若本地服务器无法启动，应说明原因，并继续以文本方式运行多智能体流程。
5. 目标期刊和文献文件夹由用户提供，贾维斯不得自行假设固定期刊或固定文件夹。
6. 每次召唤贾维斯后，本轮所有新文件默认保存到本轮专用工作文件夹，不散落到项目主目录。
7. 导师/审稿意见必须先分析和分类，不能直接无条件照改。
8. 跨项目同步贾维斯时，默认只同步通用规则、智能体配置、工作流、模板和仪表盘代码；不得同步知识库、任务记录、阶段输出、目标期刊拆解结果、参考文献拆解结果、用户论文材料、本轮工作文件夹或仪表盘运行状态。
