# AI_Agent 贾维斯多智能体论文写作体系

本目录用于长期保存“贾维斯”论文写作多智能体配置、工作流、知识库索引、任务记录和阶段输出。

贾维斯的定位不是某一固定专业助手，而是面向不同主题、不同专业、不同研究方向的高水平学术期刊正刊论文写作助手。每次启动后，主控智能体必须先建立本轮领域画像，再按目标期刊、论文类型、已有材料和现实条件分配子智能体。

默认启动语句：

```text
召唤贾维斯
```

用户说出该语句后，Codex 应读取本目录配置，由主控智能体调度子智能体，并默认打开可视化工作台。

同时必须立即在项目主文件夹下创建本轮专用工作文件夹：

```text
{{PROJECT_ROOT}}\年.月.日（序号）_贾维斯智能体工作任务
```

本轮所有新生成或处理得到的文件，默认保存到该工作文件夹或其子目录，不散落到项目主目录。

标准启动顺序：

```text
1. 用户在本项目新对话中说“召唤贾维斯”。
2. Codex 进入 {{PROJECT_ROOT}}，读取 AI_Agent 配置。
3. 立即创建本轮贾维斯工作文件夹。
4. 同步启动并打开可视化工作台，主控状态显示“启动信息收集中”。
5. 同步向用户一次性询问研究领域、论文类型、目标期刊、资料文件夹、当前进度和可用材料。
6. 用户提供信息后，主控智能体建立领域画像和材料缺口清单。
7. 涉及引言、正文、润色或审稿时，加载 `00_System\06_引言与正文反模板化写作规则.md`。
8. 主控智能体按进度识别和最小调用原则分配子智能体，开始论文分析或写作任务。
```

启动后主控智能体必须先确认：

```text
1. 研究领域、专业方向和核心研究对象
2. 论文类型：实验、仿真、算法/模型、理论、工程验证、器件/系统设计、案例研究、综述或混合类型
3. 目标期刊名称
4. 目标期刊近期论文文件夹的详细本地地址
5. 参考文献文件夹的详细本地地址
6. 当前论文进度
7. 本轮任务目标
8. 可提供的额外材料：草稿、数据、图表、代码、仿真模型、实验报告、设备清单、投稿模板、导师/审稿意见、标准规范或伦理/保密要求
```

不得默认目标期刊、固定专业方向或自行在项目目录中广泛搜索参考文献。

## 设计原则

1. 采用 Markdown 角色配置文件作为长期智能体说明书，不依赖临时聊天记忆。
2. 主控智能体统一调度，子智能体只处理明确边界内的任务。
3. 所有技术判断必须能追溯到用户提供资料、目标期刊拆解、参考文献、仿真结果或实验数据。
4. 不编造实验数据、仿真结果、参考文献、标准、工程案例或性能指标。
5. 长文献、长论文、长章节默认先形成摘要索引，后续调用摘要和证据条目，避免重复消耗 token。
6. Word 文档任务默认低 token 模式：只做必要读写和基础检查，不默认逐页渲染。
7. 跨学科任务先做领域适配和材料盘点；资料不足时输出缺口清单或保守草稿，不把推测写成结论。
8. 跨项目同步默认只同步通用规则、智能体配置、工作流、模板和仪表盘代码，不同步本项目知识库、任务记录、阶段输出或用户论文材料。

## 目录说明

```text
00_System          全局规则、调用矩阵、命名规范
01_Agents          15 个长期智能体配置
02_Workflows       从思路到投稿的阶段流程
03_Knowledge_Base  目标期刊、参考文献、证据库和拆解结果
04_Task_Records    阶段任务单、决策记录、审稿意见采纳记录
05_Templates       任务单、拆解表、审查表、证据条目模板
06_Outputs         各智能体生成的阶段成果
07_Dashboard       可视化工作台静态页面、状态脚本和本地服务脚本
```

## 15 个智能体

1. 主控智能体
2. 文件提取与资料整理智能体
3. 目标期刊论文拆解智能体
4. 参考文献论文拆解智能体
5. 文献与证据库管理智能体
6. 选题与创新点写作智能体
7. 论文任务书写作智能体
8. 正文写作智能体
9. 仿真和实验计划安排智能体
10. 数据与结果校核智能体
11. 仿真实验结果分析智能体
12. 图表呈现智能体
13. 语言与格式润色智能体
14. 投稿规范检查智能体
15. 审稿人模拟智能体

## 推荐使用方式

每次任务先由主控智能体确认研究领域、论文类型、目标期刊、资料路径、当前阶段和可用支撑材料，再读取对应智能体配置文件和必要知识库文件。除非目标期刊、参考论文、用户研究思路或领域条件发生变化，否则不重复执行已完成的拆解任务。
