# 选题可行性门禁审查模板

用于选题与创新点阶段。若审查不通过，主控智能体必须停止后续写作流程。

```text
审查阶段：选题与创新点可行性门禁
审稿结论：通过 / 修改后通过 / 暂缓推进 / 阻塞
是否进入下一阶段：是 / 否

一、选题基本判断
1. 研究问题是否具体：
2. 是否适合目标期刊：
3. 是否能从参考文献不足中推出：

二、创新点判断
1. 创新点是否具体：
2. 与已有研究的实质差异：
3. 是否存在过度声称：

三、可验证性判断
1. 是否有可执行仿真方案：
2. 是否有可执行实验方案：
3. 是否有可获得数据或设备：
4. 是否能形成证据闭环：

四、算法现场可用性
1. 算法复杂度是否可接受：
2. 是否依赖过多硬性条件：
3. 是否需要大量标注数据、特殊先验、严格同步或高精度标定：
4. 是否具备在线或准在线应用可能：
5. 与简单方法相比是否有足够工程收益：

五、工程可行性与意义
1. 工程场景是否明确：
2. 方法是否具备工程落地可能：
3. 工程意义是否有支撑：

六、阻塞条件
1. 不可行原因：
2. 缺失条件：
3. 继续推进的风险：

七、建议
1. 可补救路径：
2. 可替代研究方向：
3. 是否建议继续：
```

若“是否进入下一阶段”为“否”，不得继续生成论文任务书、正文、仿真计划或全文润色。
